# 百錬自得（Forge_in_Fire） - 開発ロードマップ

> 剣道の稽古・技の習熟度を記録し、成長をゲーミフィケーションで可視化するWebアプリ。

---

## ✅ 完了済みのフェーズ

### Phase 1〜3: 基盤構築
- Next.js (App Router) + Tailwind CSS + GAS/Spreadsheet 連携
- 基本的なCRUD、認証、レベル/XP計算ロジックの実装

### Phase 4: 評価ログの完全正規化
- `logs` テーブルの `task_id` (UUID) 化
- スマート差分検知による課題の自動アーカイブ機能

### Phase 5: スキルグリッド（サイバー八卦陣）の実装
- `@xyflow/react` を用いた六角形ノードの習熟度ビジュアライザ

### Phase 6: 実績システムとキャッシュ化
- SWR導入による画面表示の高速化（GETキャッシュ）
- ユーザーの継続を促すアチーブメント機能の実装

### Phase 7: PWA化と他者評価システムの拡張
- `@ducanh2912/next-pwa` によるオフライン対応・アプリ化
- `evaluatePeer` の課題単位配列化による一括評価対応

### Phase 8: 技習熟度の量×質マトリックス化とUI最適化
- `user_techniques` の拡張
- モバイルUXを考慮したボトムナビゲーションの導入

### Phase 9〜11: 称号・鏡映分析・免許皆伝
- **称号システム:** 3層構造（二つ名・部位称号・レベル称号）の実装。
- **免許皆伝（Mastery）:** 安定率に基づいた習熟ステータス動的判定と演出。
- **鏡映分析（Insight）:** 自己評価と他者評価の差分をダブルバーとインサイト数式で可視化。
- **HUD化（11.1）:** 設定アイコンの配置最適化とグラフ表示バグの完全修正。

## 🚀 今後の拡張ポイント（Next Phases）

### Phase 12: PWAプッシュ通知の実装と技術検証（予定）
- **PWAプッシュ通知の実装:**
  - **目的:** 「通知が来ること」をアプリを開くスイッチにし、アクティブ率を向上させる。
  - **技術検証:** Service Worker / Push API を用いた通知送受信。
  - **ノイズ対策:** 評価のたびに通知するのではなく、1日1回（21時など）に通知をまとめるためのフラグ管理を GAS 上に実装。

### Phase 13: エンドコンテンツ：反射神経養成（予定）
- **ミニゲーム「刹那の太刀」:** 画面の合図に合わせてタップし、反応速度（ms）を測定。
- **稽古連携:** ミニゲームの成績に応じて少量のXPを付与、または「反射神経」スキルの可視化。
- **ランキング連動:** 反応速度の門下生内ランキングを実装。

### Phase 14: 殿堂入りとランキング機能（予定）
- **皆伝課題の殿堂入り:** 免許皆伝に到達した課題の「卒業（一括アーカイブ）」機能と履歴表示。
- **ランキング機能:** 門下生間のXPランキングや皆伝数ランキングの実装。
