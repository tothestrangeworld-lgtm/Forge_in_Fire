# 百錬自得 (Hyakuren Jitoku) - 開発ロードマップ

## 🎯 プロジェクトの現在地
- **現在のフェーズ:** Phase 10 完了 / Phase 11 へ移行準備中
- **注力テーマ:** 自己と他者の評価ギャップの可視化と「気づき」の提供、心理的安全性の向上

---

## ✅ 完了済みのフェーズ (Phase 1 〜 Phase 10)

### Phase 1〜3: コアシステムの構築
- Next.js + GAS (Google Apps Script) によるサーバーレス構成の確立
- ユーザー認証、ダッシュボードUI（XP推移、レーダーチャート）の実装
- 稽古ログの記録とレベルアップ（XP）ロジックの基盤構築

### Phase 4〜5: データ正規化とタスク管理
- DBスキーマの正規化（タスク名のID化・UUID管理）
- スマート差分更新による課題（User Tasks）の編集・アーカイブ機能
- ダークモード＆サイバー和風デザイン（グラスモーフィズム）の適用

### Phase 6〜7: モチベーション・他者評価システム
- **アチーブメント（実績バッジ）機能:** 継続日数や累計回数に基づくバッジ付与、トースト通知
- **他者評価（ピア・エバリュエーション）機能:** 道場の門下生同士で互いの課題を評価し合うシステム（配列評価対応）

### Phase 8: 技の稽古（量×質）マトリックス
- 単純な星評価から「量（1〜5）」×「質（1〜5）」の2軸評価へ刷新
- 点滴穿石、一撃必殺などの「四字熟語フィードバック（25パターン）」実装
- リアルタイム獲得XPプレビューと動的ステッパーUIの導入

### Phase 9: 称号・二つ名システムの拡張
- **称号（Title）:** レベルに応じた称号の動的導出（DBからtitleカラムを排除する完全正規化）
- **二つ名（Epithet）:** 技の傾向に基づく称号システム。N/R/SRのレア度判定と由来説明文（description）の追加

### Phase 10: 剣風相性＆マッチングシステム・UI統一 🏆
- **相性マスタ連携:** 自分の得意技（BaseStyle）と他者の技の相性（S/W）を判定するバックエンド実装
- **サイバー和風「剣風書」ポップアップ:** 相性タグをタップすると、理由・対策・そのスタイルを持つ門下生を掲示するPortalモーダル（☆評価付き）
- **カラーパレットの洗練:** 苦手な相手を「黄色（警戒）→赤（危険）→紫（致命的脅威）」で直感的に表現
- **UIコンポーネントの共通化（引き算の美学）:** 自己評価画面と他者評価画面のUIを `TaskEvalCard` として統一し、モバイルでの視認性と操作性を極限まで向上

## 🚀 次のフェーズ (Next Steps)

### Phase 11: 「気づき」とギャップ分析のUI実装
**目的:** 自分が思う自分と、他者から見た自分の「ズレ」を可視化し、成長のヒント（インサイト）を与える。
- [ ] **ギャップ計算ロジック:** 自己評価スコアの平均と、他者評価スコアの平均の差分を計算。
- [ ] **インサイト・メッセージ機能:** 「剣友はあなたが思う以上に評価しています」「自己評価が少し厳しすぎるかもしれません」などのテキストをダッシュボードに掲示。
- [ ] **UIへの統合:** レーダーチャートや課題別スコア分布のセクションに、ギャップを示すアイコンやコメントを上品に配置。

### Phase 12: 評価の「完全匿名化」と心理的安全性の担保
**目的:** 門下生同士が忖度なく、純粋に剣道の上達のための評価を行えるようにする。
- [ ] **GASレスポンスの修正:** peerLogs などを取得する際、評価者の `name` や `user_id` を隠蔽（ハッシュ化、または一律で「ある剣士」等に変更）。
- [ ] **フロントエンドの表記変更:** 「〇〇さんからの評価」という表示を廃止し、UIデザインを匿名仕様に適合させる。
- [ ] **不正防止ロジック:** 同一ユーザーによる連続評価のブロックなど、匿名化に伴う悪用防止策の強化。

### Phase 13: PWA拡張・プッシュ通知による習慣化
**目的:** アプリの起動率（アクティベーション）を高め、毎日の稽古・記録を自然な習慣にする。
- [ ] **PWA（Progressive Web App）化:** manifest.json の整備、オフラインキャッシュ、ホーム画面への追加対応。
- [ ] **リマインダー通知:** 「今日の稽古は終わりましたか？」「〇〇さんの課題を評価しましょう」といったプッシュ通知（またはLINE Notify等の外部API連携）。

## 🔮 将来の構想 (Future Backlog)
- **AI道場破り（Bot評価システム）:** 評価が集まりにくいユーザーに対して、AIがこれまでの傾向から擬似的な評価やアドバイスを行う機能。
- **動画解析連携:** ユーザーがアップロードした稽古動画から、姿勢や竹刀の軌道をAIが解析し、自動でXPを付与する究極の記録機能。
- **大会/昇段審査モード:** 一定期間ごとに「審査」イベントを発生させ、特別なアチーブメントやXP倍率を提供するゲームニフィケーション。
