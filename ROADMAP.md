## Phase 5: コア体験の極限昇華（UI/UXの大幅強化） ✅ 完了

アプリの「顔」であるスキルグリッドを中心に、ユーザーが思わずシェアしたくなるような圧倒的なビジュアルと操作性を実装する。

### スキルグリッド（サイバー八卦陣）の見栄え向上

- [x] **カスタムノードの導入（六角形）**
  - 単なる円形ノードから `clip-path: polygon(...)` を用いた**六角形（Pointy-top Hexagon）**に変更。
  - COREノード（中心）はさらに格上の**八角形**を採用し、視覚的ヒエラルキーを強調。
  - `BodyPartNode`・`TechniqueNode` それぞれに六角形を適用。

- [x] **視覚的キャップ（上限）の実装**
  - 技のポイント（`points`）は無制限に蓄積される仕様を維持しつつ、UIの破綻を防ぐため**視覚的キャップ（`TECH_SCORE_CAP = 10`）**を導入。
  - `techNorm = Math.min(points / TECH_SCORE_CAP, 1.0)` でノードサイズ・エフェクトの最大値を固定。
  - 部位ノード（`BodyPartNode`）にも `BP_SCORE_CAP = 50` を適用。

- [x] **カンスト到達エフェクト（黄金パルス発光）**
  - `score ≥ TECH_SCORE_CAP` のノードに `@keyframes maxed-pulse` による黄金色の明滅アニメーションを追加。
  - カンスト技の数をグリッド上部に「🏆 MAX到達: N技」として表示。
  - カンスト技には `MAX` バッジを付与。

- [x] **得意技（シグネチャーテクニック）の強調**
  - `signatureTechId` と一致するノードに `@keyframes signature-pulse`（大型黄金オーラ）を適用。
  - 一回り大きいサイズ・金色グラデーション背景・★バッジで一目で識別可能。
  - グリッド上部に「得意技: ○○ Npt」バナーを表示。

- [x] **サイバーエッジ（線）のアニメーション化**
  - 全エッジに `animated: true` を付与し、データが流れるような電子回路演出を実現。
  - `stroke-dasharray + @keyframes dashmove` でダッシュが流れるサイバーアニメーション。
  - シグネチャー技・カンスト技へのエッジは金色 + グロー発光（`drop-shadow` フィルター）。
  - 習熟度に応じたエッジ色（金 → 紫 → 深紫 → 半透明）で状態を可視化。

### 全体アニメーションの洗練

- [ ] ページ遷移時やデータ保存完了時のスタイリッシュなマイクロインタラクション。

## Phase 6: 体感速度の極限最適化（フロントエンド軽量化） 👈 NEXT!

GASのレスポンス待ちによる画面表示の遅延を解消し、ユーザーが「速い」と体感できるフロントエンドを構築するフェーズ。  
後続の PWA 化（Phase 7）の土台となる通信設計をここで確立する。

### データキャッシュ戦略の導入

- [ ] **SWR または React Query の導入**
  - GAS エンドポイントへの fetch を SWR / React Query でラップし、`stale-while-revalidate` 戦略によるキャッシュを実装。
  - 初回表示はキャッシュデータを即座に描画し、バックグラウンドで最新データを取得・差分更新。
  - キャッシュキーの設計（ユーザーID・シート種別ごと）とキャッシュ有効期限の調整。
  - ページ遷移時の再フェッチ抑制（`revalidateOnFocus` / `staleTime` のチューニング）。

### 楽観的UI（Optimistic UI）の導入

- [ ] **保存・更新操作の即時フィードバック**
  - 稽古ログの保存・技スコアの更新などユーザー操作を、GASレスポンスを待たずにローカルステートへ即時反映。
  - GASからエラーレスポンスが返った場合はロールバック処理とトースト通知でユーザーに伝達。
  - 楽観的更新対象: ログ保存、他者評価、シグネチャー技の変更、タスク更新。

### その他の軽量化施策

- [ ] 不要な再レンダリングの削減（`React.memo` / `useMemo` / `useCallback` の適用見直し）。
- [ ] スキルグリッドなど重量コンポーネントの遅延ロード（`dynamic import` + `Suspense`）。
- [ ] GAS 側のレスポンス最小化（不要フィールドの除外・シート読み取り範囲の最適化）。

## Phase 7: PWA（Progressive Web App）化

**Phase 6 でフロントエンドの通信を最適化した後に実施する。**  
アプリをネイティブアプリのようにスマートフォンのホーム画面から即起動できるようにし、オフライン環境でも基本機能を利用可能にするフェーズ。

### ホーム画面インストール対応

- [ ] **`manifest.json` の整備**
  - `name`・`short_name`・`theme_color`・`background_color`・`display: standalone` を設定。
  - スプラッシュスクリーン用の各種解像度アイコン（192px / 512px / maskable）を作成。
  - サイバー和風テイストを体現したアプリアイコンのデザイン（剣・八卦・回路モチーフの融合）。

### Service Worker の導入

- [ ] **`next-pwa` の導入とキャッシュ戦略の設定**
  - `next-pwa`（または `@ducanh2912/next-pwa`）を導入し、静的アセット（JS・CSS・フォント・画像）をキャッシュ。
  - オフライン起動時はキャッシュ済みページを表示し、「オフラインモード」であることをバナーで明示。
  - ネットワーク復帰時にバックグラウンドでキャッシュを自動更新（Background Sync）。

### オフライン記録 → 同期フロー

- [ ] ネットワーク不在時に稽古ログをローカル（IndexedDB / localStorage）へ一時保存。
- [ ] オンライン復帰後に未同期ログを GAS へ自動アップロード（同期状態をインジケーターで表示）。

## Phase 8: エンドコンテンツ（反射神経ミニゲーム）

アプリの最終到達点として、剣道の「出端（でばな）」や「起こり」を捉える瞬発力をゲームで鍛えるフェーズ。  
稽古記録・スキル育成で積み上げた実力を、遊びの中でも試せるエンドコンテンツを提供する。

### ゲーム設計

- [ ] **リアクションタイム計測ゲーム**
  - ランダムなインターバル後に「起こり（攻撃の予兆）」を示す視覚・音響シグナルを表示。
  - プレイヤーは「面」「小手」「胴」「突き」など技を選んでタップ（またはキー入力）し、最適な技の選択精度とリアクションタイムの両方を評価。
  - 難易度設定: 初伝（低速・シグナル明瞭）→ 中伝 → 奥伝（高速・フェイント混じり）。
  - プレイ結果（平均反応時間・精度・段位別ランク）をセッションごとに記録。

### 報酬システム

- [ ] **スコアに応じた XP 付与**
  - ゲームのスコア（反応速度 × 正解率 × 難易度係数）を XP に換算し、通常の稽古ログと同じ成長システムに統合。
  - 1日あたりの XP 獲得上限を設け、ゲームプレイが稽古記録の代替にならないよう設計。

- [ ] **専用の二つ名・称号バッジ**
  - ゲーム実績に応じた特別称号を付与（例：「電光の剣士」「残影」「見切り名人」など）。
  - 称号はプロフィール・ライバル閲覧画面に表示し、他者への差別化要素とする。
  - 累計プレイ回数・自己ベスト更新などのマイルストーンでアチーブメントバッジを解放。

### 技術実装

- [ ] Canvas または CSS アニメーションを用いたゲーム画面の実装（Next.js ページとして組み込み）。
- [ ] ゲーム結果を `minigame_scores` シート（GAS）へ記録・集計。
- [ ] グローバルリーダーボードの表示（ランキング画面と連携）。
